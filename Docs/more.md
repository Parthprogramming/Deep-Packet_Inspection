Your engine detects DNS tunneling purely on packet size + port number. It doesn't parse DNS record types or validate the query structure. That's a deliberate simplification — real production IDS (Suricata, Zeek) go deeper and check entropy of the subdomain label, query frequency per domain, and response sizes. Your engine catches the obvious case: any DNS packet over 512 bytes is already suspicious by the DNS spec (RFC 1035 says standard UDP DNS must be under 512 bytes without EDNS0 extensions).
